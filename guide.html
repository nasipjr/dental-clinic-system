1. نظرة عامة على النظام

نظام DCMS هو تطبيق ويب متكامل مبني على Flask لإدارة عيادة أسنان بالكامل. يتيح للطاقم الطبي إدارة المرضى، المواعيد، المعالجات، الفواتير، والمدفوعات من مكان واحد.

👤

إدارة المرضى

ملف شامل: بيانات شخصية، طبية، مالية، ملفات X-ray

📅

إدارة المواعيد

جدولة، تعديل، إلغاء، تقويم تفاعلي

🩺

جلسات المعالجة

تسجيل الإجراءات الطبية لكل موعد

🧾

الفواتير

إنشاء وإدارة مع خصم ثابت أو نسبة مئوية

💰

المدفوعات

توزيع تلقائي FIFO على الفواتير الأقدم

📊

التقارير

رسوم بيانية للإيرادات وإحصائيات شاملة

🌐

بوابة المريض

مدخل خاص للمريض للحجز والمتابعة

🔔

الإشعارات

تذكيرات SMS/WhatsApp/Email تلقائية

🌙

الوضع الليلي

Dark/Light mode محفوظ في localStorage

🌍

ثنائية اللغة

عربي وإنجليزي مع دعم RTL كامل

💾

النسخ الاحتياطية

تلقائية يومية + يدوية (mysqldump)

📱

PWA

قابل للتثبيت كتطبيق عبر service worker

⚠️
أول تسجيل دخول: حساب admin الافتراضي هو admin / admin123 — غيّر كلمة المرور فوراً!

تسلسل التهيئة (أول تشغيل)

1

db.create_all()

إنشاء جميع جداول قاعدة البيانات

2

populate_default_settings()

تعبئة الإعدادات الافتراضية (اسم العيادة، ساعات العمل...)

3

ترحيلات تلقائية (5 دوال)

إضافة أعمدة جديدة إن لم تكن موجودة (discount, tax_rate, session_opened_at...)

4

ensure_default_admin()

إنشاء حساب admin افتراضي إن لم يوجد أي admin

5

تشغيل Background Threads

خيط التذكيرات (كل 5 دقائق) + خيط النسخ الاحتياطية (كل 24 ساعة)

8. طبقة الخدمات (Services)

invoice_service.py

# إذا له معالجات → أنشئ/احتفظ بالفاتورة
# إذا لا معالجات → احذف الفاتورة تلقائياً

payment_service.py — خوارزمية FIFO

allocate_patient_payments_to_invoices(patient_id)
# 1. احذف كل PaymentAllocation الحالية للمريض
# 2. رتّب الدفعات من الأقدم للأحدث
# 3. رتّب الفواتير من الأقدم للأحدث

#    لكل فاتورة غير مدفوعة:
#      خصص ما يكفي لتغطية المستحق
#      أنشئ PaymentAllocation
# 5. flush() — لا commit (الـ caller يفعل ذلك)

11. نظام الإشعارات التلقائية

آلية عمل خيط التذكيرات

كل 5 دقائق ← process_reminders()
    → تجلب المواعيد الـ Scheduled في الـ 25 ساعة القادمة
    → لكل موعد:
        إذا 22-25 ساعة → تذكير "24h" (SMS + WhatsApp + Email)
        إذا 1.5-2.5 ساعة → تذكير "2h"
    → يتحقق من NotificationLog (لا يُعيد الإرسال)
القناةالمزودالشرط
SMSTwilio RESTphone موجود + enable_sms=true
WhatsAppTwilio WhatsAppphone موجود + enable_whatsapp=true
EmailSMTP Gmail (port 587)email موجود + enable_email=true
✅
وضع المحاكاة: إذا لم تُضبط بيانات Twilio/SMTP، يكتب النظام في السجل [MOCK SMS] بدون إرسال فعلي.

15. تدفقات العمل الكاملة

1️⃣ الجلسة الطبية الكاملة

1

فتح الجلسة

Doctor → Appointments → session_opened_at = now()

2

إضافة معالجات

اختيار الإجراء + رقم السن + التكلفة → تُنشأ الفاتورة تلقائياً عند أول معالجة

3

إنهاء الجلسة

End Session → status = "Done"

4

استلام الدفعة

تسجيل دفعة → توزيع FIFO تلقائي على الفواتير

2️⃣ حجز موعد من البوابة

1

المريض يختار الوقت

من تقويم البوابة → يرى الفترات المتاحة فقط

2

يرسل الطلب

Appointment(status="Pending") — يظهر badge أحمر للكادر

3

الكادر يوافق أو يرفض

موافقة → Scheduled / رفض → Cancelled

3️⃣ فاتورة يدوية (Walk-in)

1

Invoices → Add Invoice

اختيار مريض + تاريخ الزيارة

2

إضافة بنود المعالجة

صفوف متعددة + خصم اختياري

3

اختيار الدفع

بلا دفعة / كاملة الآن / مبلغ مخصص

4

النظام ينشئ تلقائياً

Appointment(Done) + Treatments + Invoice + Payment (اختياري)
